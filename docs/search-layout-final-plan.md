# BrewMap Search Layout Final Plan

## 결정

BrewMap의 첫 화면을 히어로 중심 랜딩 화면에서 검색 결과 중심 화면으로 전환한다.

Yelp처럼 사용자가 익숙한 지역 검색 서비스의 정보 구조는 참고하되, 화면의 판단 기준은 별점, 리뷰, 광고가 아니라 BrewMap의 핵심 가치인 "마시고 싶은 커피가 실제로 있는지"와 "그 정보가 얼마나 검증되었는지"에 둔다.

## 목표 화면 구조

- 상단 고정 검색 바: 검색어, 지역, 주요 이동 메뉴
- 좌측 필터 패널: 커피 태그, 권역, 신뢰도, 검증 상태
- 중앙 결과 리스트: 카페 카드, 가능 커피, 최근 확인일, 저장/상세/지도 링크
- 우측 지도 패널: 결과 핀, 클러스터, 현재 결과 수, 내 위치
- 레트로 데스크톱 앱 톤: 메뉴바, 창 프레임, 베벨 버튼, 격자형 배경으로 앱 셸을 고정
- 모바일 화면: 목록, 지도, 필터를 탭처럼 전환할 수 있는 단일 열 구조

## Yelp와 구별할 차별점

- 별점/리뷰를 노출하지 않는다.
- 결과 카드의 첫 번째 판단 기준은 커피 가능 여부다.
- `신뢰도`, `최근 확인일`, `검증 출처`를 카드 상단 정보로 노출한다.
- "제보"와 "관리자 검증 큐"를 제품 흐름에 포함한다.
- 지역 확장은 부산 우선, 서울 확장 준비라는 MVP 범위 안에서 다룬다.

## 레트로 데스크톱 앱 디자인

디자인 샘플 비교 영역은 제거하고, 첫 화면 자체를 레트로 데스크톱 앱 스타일로 고정한다.

사용자는 별도 프리뷰를 전환하지 않고 실제 검색, 필터, 결과, 지도, 저장/제보/Admin 흐름을 같은 앱 셸 안에서 확인한다. 시각 톤은 메뉴바, 창 프레임, 베벨 버튼, 격자형 데스크톱 배경을 사용하되 커피 가능 여부와 검증 정보의 위계는 유지한다.

## 구현 계획

1. 문서화
   - 최종 화면 방향, 차별점, 구현 순서, 완료 기준을 기록한다.
   - README의 주요 문서 목록에 본 문서를 추가한다.

2. HTML 구조 개편
   - 기존 히어로 섹션을 검색 앱 셸로 전환한다.
   - 검색, 필터, 결과 리스트, 지도 패널이 한 화면에 들어오도록 마크업을 재배치한다.
   - 저장/제보/Admin 영역은 하단 운영 섹션으로 유지한다.

3. 결과 카드 개편
   - 그리드형 카드를 세로 리스트형 카드로 바꾼다.
   - 카페명, 권역, 신뢰도, 최근 확인일, 검증 출처, 가능 커피 태그를 빠르게 스캔할 수 있게 정리한다.
   - 저장, 상세, 네이버/카카오/Google Maps 링크아웃 액션은 유지한다.

4. CSS/반응형 정리
   - 데스크톱은 `필터 / 결과 / 지도` 3영역 레이아웃을 사용한다.
   - 지도는 데스크톱에서 sticky 패널로 유지한다.
   - 모바일은 지도와 필터가 목록 아래로 자연스럽게 흐르도록 단일 열로 전환한다.
   - BrewMap의 색과 톤은 유지하되, 카드형 랜딩 페이지 느낌은 줄인다.

5. 레트로 데스크톱 앱 디자인
   - 상단 메뉴, 검색 영역, 필터, 결과, 지도, 운영 섹션을 창 프레임처럼 보이게 정리한다.
   - 버튼, 입력, 상태 배지는 베벨과 단단한 외곽선을 사용해 데스크톱 앱 조작감으로 맞춘다.
   - 샘플 비교 메뉴와 정적 프리뷰 데이터는 제거한다.

6. 검증
   - `npm run lint`
   - `npm run build`
   - `npm run data:check`
   - 로컬 서버로 첫 화면, 검색, 필터, 저장, 상세, 지도 렌더링을 확인한다.

## 완료 기준

- 첫 화면에서 검색, 필터, 결과, 지도가 동시에 인지된다.
- 카페 결과가 Yelp의 리뷰 중심 카드가 아니라 BrewMap의 커피 가능 여부 중심 카드로 보인다.
- 첫 화면이 레트로 데스크톱 앱 셸로 보이고 별도 디자인 샘플 메뉴가 노출되지 않는다.
- 기존 P0 기능인 지도 핀, 필터, 상세, 저장, 제보, Admin 기능이 깨지지 않는다.
- 모바일 폭에서 텍스트와 버튼이 겹치지 않는다.
- 빌드와 검증 스크립트가 통과한다.
