# BrewMap Retro UI/UX 점검 체크리스트

기준 문서: `docs/brewmap_retro_desktop_spec.md`  
작성일: 2026-06-24  
목적: 기획서 기준과 현재 동작/구현 방식의 차이를 확인하고, 기획 의도대로 수정한 뒤 완료 여부를 체크한다.

## 기획서 업데이트

- [x] `NEARBY_MAP.EXE`가 기획서 작성 이후 추가된 구성임을 명시한다.
- [x] `BREWMAP.EXE`는 선택 카페 중심 약도, `NEARBY_MAP.EXE`는 주변 일반 지도로 역할을 분리한다.
- [x] 모바일 작업 표시줄과 프로그램 목록 기준을 4개에서 5개 프로그램으로 갱신한다.

## 기획서 기준 vs 현재 구현 차이

| 항목 | 기획서 기준 | 확인한 현재 차이 | 기대 동작/수정 기준 | 완료 |
|---|---|---|---|---|
| 프로그램 구성 | 다섯 프로그램: `LOCAL_ZINE`, `CAFE_INDEX`, `BREWMAP`, `NEARBY_MAP`, `BREW_LOG` | 구현에는 `NEARBY_MAP`이 있으나 기획서와 구현 메모는 4개 프로그램 기준으로 남아 있었다. | 문서와 구현 메모를 5개 프로그램 기준으로 맞춘다. | - [x] |
| 최초 진입 | `LOCAL_ZINE.EXE`만 열고 전면 표시 | 초기 상태와 Reset Layout이 `CAFE_INDEX.EXE`까지 열고 활성화했다. | 초기 상태와 Reset Layout 모두 `LOCAL_ZINE.EXE` 단일 창으로 시작한다. | - [x] |
| 상단 메뉴 | 레트로 셸 안에서 프로그램을 열고 전환 | 상단 메뉴가 `#legacy-home`, `#map` 등 레거시 앵커로 스크롤했다. | 상단 메뉴는 레거시 스크롤 없이 레트로 프로그램만 연다. | - [x] |
| 레거시 DOM 노출 | 레트로 기본 화면은 100dvh 안에서 독립 동작 | `syncRoute()`가 레거시 DOM을 다시 표시해 화면 아래 기존 UI가 노출될 수 있었다. | 레거시 DOM은 유지하되 `hidden`/`aria-hidden` 상태로 둔다. | - [x] |
| 지도 역할 | `BREWMAP`과 `NEARBY_MAP` 역할 분리 | 구현은 분리되어 있으나 문서 기준이 누락되어 검증 기준이 불명확했다. | `BREWMAP`은 선택 카페 약도, `NEARBY_MAP`은 일반 주변 지도 기준으로 검증한다. | - [x] |
| 모바일 작업 표시줄 | 5개 프로그램을 좁은 화면에서도 구분 | `NEARBY_MAP`에 모바일 라벨이 없어 긴 라벨이 노출될 수 있었다. | `NEARBY_MAP` 모바일 라벨을 `주변`으로 지정한다. | - [x] |
| 배포 캐시 | 수정된 CSS/JS가 즉시 갱신 | HTML과 서비스워커 asset version이 과거 값으로 남아 있었다. | HTML 쿼리와 서비스워커 캐시 버전을 2026-06-24 기준으로 올린다. | - [x] |
| 자동 검증 | 검증 스크립트가 기대 동작을 보호 | 기존 검증은 레거시 DOM 노출을 기대하는 방향이었다. | 검증 기준을 레거시 숨김, 레트로 프로그램 메뉴, `NEARBY_MAP` 포함으로 바꾼다. | - [x] |
