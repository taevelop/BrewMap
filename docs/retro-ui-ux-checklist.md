# BrewMap Retro UI/UX 점검 체크리스트

작업 전 기존 검색/지도/저장 흐름과 레트로 데스크톱 전환 후 동작을 비교하기 위한 체크리스트입니다.

## 점검 항목

- [x] 레트로 진입 상태에서 기존 테스트/레거시 UI가 스크롤 하단에 노출되지 않는다.
- [x] 레트로 상단 메뉴와 작업표시줄이 기존 핵심 흐름(탐색, 검색/필터, 지도, 저장)을 레트로 프로그램 안에서 수행한다.
- [x] BREWMAP.EXE는 선택 카페 중심의 약도 역할로 분리한다.
- [x] NEARBY_MAP.EXE는 내 위치/현위치 기준 일반 지도 역할로 분리한다.
- [x] 주변 일반 지도 프로그램은 실제 지도 타일, 핀, 확대/축소, 드래그 이동, 카페 선택을 지원한다.
- [x] 카페 선택/저장 상태가 로컬진, 인덱스, 지도, 장부 프로그램 사이에서 일관되게 갱신된다.
- [x] 모바일 화면에서는 데스크톱 창이 화면 밖으로 밀리지 않고 활성 프로그램이 전체 화면 카드처럼 표시된다.
- [x] 작은 화면에서 지도/인덱스/장부 내부 컬럼이 단일 컬럼으로 접히고 터치 가능한 버튼 크기를 유지한다.
- [x] 창 이동/리사이즈 이후에도 현재 화면 크기 안에 창 위치를 보정한다.
- [x] 레트로 UI는 100dvh 안에서 자체 스크롤을 사용하여 페이지 전체 스크롤로 레거시 영역에 도달하지 않는다.
