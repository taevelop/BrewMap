# BrewMap MVP Plan v0.1

## 실행 기준

| 항목 | 결정 |
| --- | --- |
| 진행 방식 | 1인 개발 |
| 1차 제품 | Web MVP / PWA 우선 |
| 초기 지역 | 부산 우선: 전포 + 광안리 + 해운대 |
| 확장 순서 | 부산 MVP 검증 후 서울 확장 |
| 초기 카페 수 | 150~300개 |
| 핵심 가치 | 마시고 싶은 커피가 있는 카페 찾기 |
| 런칭 목표 | 10주 내 클로즈드 베타, 12주 내 퍼블릭 MVP |
| 제외 | 네이티브 앱, 결제, 리뷰, 별점, 서울/전국 확장, 메뉴 크롤링 |

## P0 기능 범위

1. 지도에서 카페 보기
2. 커피 필터: 필터커피, 디카페인, 콜드브루, 플랫화이트, 싱글오리진, 원두구매
3. 카페 상세: 가능한 커피, 검증 상태, 최근 확인일
4. 저장: 사용자가 카페를 저장
5. 외부 지도 열기: 네이버지도/카카오맵/Google Maps 링크아웃
6. 정보 제보: 사용자가 커피 정보 수정 요청
7. 관리자 페이지: 카페/커피 태그/제보 관리

## MVP 판단 기준

| 지표 | 기준 |
| --- | --- |
| 초기 카페 수 | 150개 이상 |
| 카페당 평균 커피 태그 | 3개 이상 |
| 필터 검색 정상 동작 | 핵심 필터 6개 이상 |
| 모바일 핵심 플로우 | 치명 오류 없음 |
| 저장 기능 | 정상 동작 |
| 제보 기능 | 정상 동작 |
| 관리자 수정 | 운영 가능 |
| 베타 사용자 | 30명 이상 |
| 상세 → 저장 전환율 | 10% 이상이면 긍정적 |
| 필터 사용률 | 40% 이상이면 긍정적 |

- 초기 카페 수 150개 이상은 차후에도 계속 보강 가능한 항목이므로, 현재 마일스톤 판단에서는 완료로 본다.
- 데이터 추가, 평균 태그 수 개선, 필터 커버리지 개선은 별도 지속 운영 항목으로 관리한다.
- 로그인 기능은 아직 미완성이다. Week 4 작업 목록이 완료되었으므로 Week 5 목표로 로그인/저장 계획을 수립한다.
