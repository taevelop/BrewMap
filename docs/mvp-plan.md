# BrewMap MVP Plan v0.1

## 실행 기준

| 항목 | 결정 |
| --- | --- |
| 진행 방식 | 1인 개발 |
| 1차 제품 | Web MVP / PWA 우선 |
| 초기 지역 | 성수 + 연남 우선 |
| 초기 카페 수 | 150~300개 |
| 핵심 가치 | 마시고 싶은 커피가 있는 카페 찾기 |
| 런칭 목표 | 10주 내 클로즈드 베타, 12주 내 퍼블릭 MVP |
| 제외 | 네이티브 앱, 결제, 리뷰, 별점, 전국 확장, 메뉴 크롤링 |

## P0 기능 범위

1. 지도에서 카페 보기
2. 커피 필터: 필터커피, 디카페인, 콜드브루, 플랫화이트, 싱글오리진, 원두구매
3. 카페 상세: 가능한 커피, 검증 상태, 최근 확인일
4. 저장: 사용자가 카페를 저장
5. 외부 지도 열기: 네이버지도/카카오맵/Google Maps 링크아웃
6. 정보 제보: 사용자가 커피 정보 수정 요청
7. 관리자 페이지: 카페/커피 태그/제보 관리

## 12주 실행 플랜

| 기간 | 목표 | 완료 기준 |
| --- | --- | --- |
| Week 1 | 기획 고정 | P0/P1/P2, 초기 지역, 태그 v1, ERD 초안, 작업 보드 |
| Week 2 | 설계/세팅 | 기술스택, DB 마이그레이션, 와이어프레임, Seed CSV |
| Week 3 | 지도/카페 조회 | 지도, 핀, bounds 조회, 카드, 샘플 50개 |
| Week 4 | 필터/상세 | 필터, 상세, 외부 지도, 필터 QA, 샘플 100개 |
| Week 5 | 로그인/저장 | 로그인, 저장/해제, 저장 리스트, 저장 QA |
| Week 6 | 제보/Admin | 제보 폼, Admin CRUD/태그/제보, CSV Import |
| Week 7 | 데이터 구축 | 카페 150개, 태그/신뢰도/확인일, 데이터 QA |
| Week 8 | 클로즈드 베타 | 베타 30명, 사용 로그, 오류/UX/데이터 수정 |
| Week 9 | 품질 개선 | 카페 250개, 모바일 UX, 성능, SEO, 약관 |
| Week 10 | 퍼블릭 준비 | 런칭 후보 빌드, 최종 QA, 랜딩, 피드백 채널 |
| Week 11~12 | 퍼블릭 오픈 | 배포, 홍보, 데이터 보강, 사장님 인증 후보, 지표 리뷰 |

## MVP 판단 기준

| 지표 | 기준 |
| --- | --- |
| 초기 카페 수 | 150개 이상 |
| 카페당 평균 커피 태그 | 3개 이상 |
| 필터 검색 정상 동작 | 핵심 필터 6개 이상 |
| 모바일 핵심 플로우 | 치명 오류 없음 |
| 저장 기능 | 정상 동작 |
| 제보 기능 | 정상 동작 |
| 관리자 수정 | 운영 가능 |
| 베타 사용자 | 30명 이상 |
| 상세 → 저장 전환율 | 10% 이상이면 긍정적 |
| 필터 사용률 | 40% 이상이면 긍정적 |
