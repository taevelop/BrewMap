# BrewMap 공유 리뷰 개선 계획

작성일: 2026-06-25  
대상 링크: https://chatgpt.com/share/6a3c8372-c5d8-83ee-81ff-0ebe048d4c10  
작업 브랜치: `codex/chatgpt-improvements-20260625`

## 1. 확인한 문제

공유 리뷰의 최종 판정은 색상이나 카드 장식보다 정보 구조가 우선이라는 내용이다. 현재 공개 화면에는 사용자 제품, 내부 명세, 운영 도구가 같은 흐름에 섞여 있다.

확인된 현재 상태:

- `index.html` 메타 제목과 설명에 `Coffee Map MVP`, `MVP`가 노출된다.
- 공개 상단 메뉴에 `Admin`이 보이고, 같은 페이지 하단에 Admin Queue, Cafe CRUD, Tag Management, CSV Import, Audit Log가 노출된다.
- 저장 영역에 "로그인 사용자는..." 형태의 PRD 인수 조건 문장이 노출된다.
- `FIND YOUR COFFEE`, `Coffee Results`, `Saved`, `Report` 등 내부 컴포넌트성 영문 라벨이 공개 UI에 섞여 있다.
- 결과 카운트가 초기 HTML에서 `0개 카페`로 시작해 로딩, 빈 결과, 오류, 초기 상태를 구분하지 못한다.
- 검색, 결과, 지도, 저장, 제보, Admin이 하나의 공개 문서 안에 연속 배치되어 핵심 과업이 흐려진다.

## 2. 개선 원칙

1. 공개 서비스와 운영 도구의 경계를 먼저 나눈다.
2. 공개 UI 카피는 사용자가 이해하는 한국어 과업 중심 문장으로 통일한다.
3. 검색 결과 상태는 로딩, 성공, 빈 결과, 오류로 분리한다.
4. 저장은 로그인 요구사항 문장이 아니라 사용자의 행동 흐름으로 설명한다.
5. 지도와 목록은 같은 카페 선택 상태를 공유하고, 지도만으로 접근해야 하는 정보를 만들지 않는다.
6. 운영 기능은 별도 관리자 진입점과 화면으로 이동하고, 실제 서버 권한 검증은 백엔드 도입 시 필수 작업으로 남긴다.

## 3. 구현 단계

### 1차: 릴리스 차단 항목

목표: 공개 페이지에서 내부 운영 정보와 개발 단계 문구를 제거한다.

- 공개 메타데이터를 `브루맵 | 부산에서 원하는 커피를 찾는 지도`와 사용자 중심 설명으로 수정한다.
- 공개 내비게이션에서 Admin 링크를 제거한다.
- 공개 페이지의 Admin Queue와 Admin Workspace를 별도 `admin.html`로 이동한다.
- 공개 저장 패널의 PRD 문장을 사용자 안내와 로그인 진입점 CTA로 바꾼다.
- `MVP`, `베타`, `서울 확장 준비`, `CRUD`, `Queue`, `Audit` 같은 내부 표현을 공개 UI에서 제거한다.
- 결과 카운트 초기값을 로딩 상태로 바꾸고, 데이터 로딩 실패와 빈 검색 결과 문구를 분리한다.

완료 기준:

- 공개 `index.html`에서 `Admin`, `MVP`, `CRUD`, `Queue`, `Audit`, `Coffee Results`, `Saved`, `Report`, `FIND YOUR COFFEE`가 사용자 화면 라벨로 노출되지 않는다.
- `admin.html`에서만 관리자 작업 영역에 진입한다.
- `npm run lint`, `npm run build`가 통과한다.

### 2차: 핵심 검색 UX

목표: 검색에서 결과 확인, 근거 확인, 저장까지 이어지는 흐름을 강화한다.

- 필터 제목을 지역, 커피, 검증 기준으로 정리한다.
- 검색어와 선택 필터를 URL 쿼리스트링에 반영한다.
- 결과 카드의 우선순위를 카페명, 지역, 제공 커피, 검증 근거, 저장/상세 액션으로 재정렬한다.
- 카페 카드 선택과 지도 핀 선택이 같은 선택 상태를 공유하도록 한다.
- 빈 결과 상태에 필터 해제, 전체 초기화, 정보 제보 액션을 제공한다.
- 상세 모달에서 검증 근거와 저장 상태를 더 명확히 보여준다.

완료 기준:

- 새로고침 후에도 검색어와 필터 상태가 복원된다.
- 결과 카드 또는 지도 핀 선택 시 같은 카페가 강조된다.
- 빈 결과, 오류, 성공 상태가 서로 다른 문구와 액션을 가진다.

### 3차: 저장, 관리자, 품질 기준

목표: 저장 플로우와 운영 화면을 사용자/운영자 관점으로 정리하고 자동 검증을 갱신한다.

- 저장 버튼 클릭 후 로그인 안내 토스트 또는 상태 메시지를 제공한다.
- 관리자 화면은 요약, 제보 검토, 카페 관리, 태그 관리, CSV 가져오기, 변경 기록 섹션으로 나눈다.
- CSV는 검증 결과와 반영 버튼이 분리되어 dry-run 흐름을 유지한다.
- 관리자 삭제/폐업 계열 액션은 확인 단계를 둔다.
- 접근성 기준을 검증 스크립트에 반영한다: 포커스 스타일, 버튼 `aria-label`, 모달 닫기, 지도 대체 목록.
- 성능 기준을 문서와 검증에 반영한다: 지도 컨테이너 고정 크기, 정적 결과 HTML 우선, 빌드 산출물 확인.

완료 기준:

- 공개 앱과 관리자 앱 모두 정적 빌드에 포함된다.
- `npm run data:check`, `npm run lint`, `npm run build`가 통과한다.
- 브라우저 런타임 또는 대체 headless 검증으로 공개 화면의 카드 렌더링과 관리자 화면 진입을 확인한다.

## 5. 백엔드 후속 조건

현재 저장소는 의존성 없는 정적 Web/PWA 구조다. 따라서 `/admin` 서버 세션 검증, 역할 기반 권한, 쓰기 API의 서버 측 권한 재검증은 이번 정적 구현만으로 완결할 수 없다. 이번 브랜치에서는 공개 노출 제거, 관리자 화면 분리, UI 안전장치, 검증 스크립트까지 완료하고, 실제 권한 통제는 서버/API 도입 시 차단 조건으로 유지한다.
